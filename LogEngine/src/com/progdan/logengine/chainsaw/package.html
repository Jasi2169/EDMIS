Chainsaw is a GUI log viewer and filter for the logengine package. By default it listens for LoggingEvent objects sent using the SocketAppender and displays them in a table. The events can be filtered based on:

  • Level
  • Thread name
  • Logger
  • Message
  • NDC

All the details for each event can be displayed by selecting the event in the table.

Chainsaw also supports loading a events logged to a file using the XMLLayout format. This is great for analysing log files, and means you do not need to keep Chainsaw running continously. It is easy to add support for loading events from other sources like JDBC.

Requirements

Chainsaw is based on the Swing API which requires JDK 1.2 or later.

Running chainsaw

Setup

You need to include the logengine.jar in the classpath.

Usage

The command line usage is:

  java -D<property>=<value> com.progdan.logengine.chainsaw.Main

The default behaviour of chainsaw can be changed by setting system properties using the -D<property>=<value> arguments to java. The following table describes what properties can be set:

Property Description
chainsaw.port Indicates which port to listen for connections on. Defaults to "4445".

Configuring Log4J

You will need to configure logengine to send logging events to Chainsaw. Here is a sample logengine.properties file for sending logging events to Chainsaw.

logengine.rootLogger=DEBUG, CHAINSAW_CLIENT

logengine.appender.CHAINSAW_CLIENT=com.progdan.logengine.net.SocketAppender
logengine.appender.CHAINSAW_CLIENT.RemoteHost=localhost
logengine.appender.CHAINSAW_CLIENT.Port=4445
logengine.appender.CHAINSAW_CLIENT.LocationInfo=true
